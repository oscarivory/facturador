User (Request Scoped) anotaciones para objetos inyectados. Un objeto que se define como @RequestScopedse crea una vez para cada solicitud y es compartido por todos los beans que lo inyectan a lo largo de una solicitud. : #{testBean.user.id} - #{testBean.user.name}

User (Conversation Scoped): estamos definiendo el bean como capaz de rastrear una conversación con un cliente. El bean mantendrá su estado hasta que termine la conversación. Mientras tanto, durante la conversación, el cliente puede ejecutar solicitudes AJAX contra el bean o navegar a otras páginas que todavía hacen referencia a este mismo bean administrado. En estos casos, el bean mantendrá su estado en las interacciones del cliente. #{conv.user.id} - #{conv.user.name}

User (Session Scoped): Especifica que un bean tiene un alcance de sesión. El contexto de la sesión se comparte entre todas las solicitudes de servlet que se producen en la misma sesión HTTP. El contexto de la sesión se destruye cuando se agota el tiempo de espera de HTTPSession, después de que se haya llamado a todos los HttpSessionListeners y al final de cualquier solicitud en la que se haya llamado invalidate () , después de que se hayan llamado todos los filtros y ServletRequestListeners . #{sess.user.id} - #{sess.user.name}

User (Stateless): EJB no existe una relación definida entre el proxy (y, por lo tanto, el cliente) y la instancia de bean real. Varios clientes pueden "compartir" una sola instancia de EJB entre invocaciones de métodos. Un cliente ansioso, por otro lado, podría sobrecargar el servidor y comunicarse con varias instancias a través del mismo proxy a la vez. Hay una relación N: M entre un cliente y las instancias de bean. El tamaño de M depende de la configuración del servidor y la carga de solicitud / transacción. Una solicitud siempre será manejada completamente por una sola instancia de bean. Varias solicitudes nunca compartirán una sola instancia simultáneamente. También es imposible para un cliente liberar una instancia o "desconectarse" o "disociarse" de ella. #{testBean.userFromStateless.id} - #{testBean.userFromStateless.name}
